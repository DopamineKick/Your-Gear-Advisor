1. ARCHITEKTURA WSPÓLNA
-----------------------

Każdy sklep korzysta z tej samej architektury:

  Import URL-i → gear_items_raw (processed = false)
           ↓
    npm run pipeline
      Krok 1: embeddingi dla już enrichowanych (cleanup)
      Krok 2: scraping unprocessed → gear_items_raw (processed = true)
      Krok 3: AI enrichment → gear_items
      Krok 4: embeddingi dla nowych gear_items

WAŻNE: Każdy pipeline wznawia od miejsca przerwania.
Można zatrzymać (Ctrl+C) i wznowić "npm run pipeline" w dowolnym momencie.

  Etap          | Dlaczego wznawia?
  --------------|--------------------------------------------------
  Scraping      | Czyta tylko rekordy z processed = false
  AI Enrichment | Czyta gear_items_raw WHERE id NOT IN gear_items
  Embeddingi    | Czyta gear_items WHERE embedding IS NULL

2. THOMANN
==========================================

Źródło: lokalny plik thomann-filter/guitar_products.csv

Import URL-i (jednorazowy, bezpiecznie powtarzalny):

Dodanie nowych produktów w przyszłości:
  1. Podmień lub rozszerz plik thomann-filter/guitar_products.csv

3. GEAR4MUSIC
==========================================

-- Metoda A: z pliku CSV (użyta przy obecnym pipeline) --

  Plik: g4m_products_wybrane.csv (~6 000 URL-i)

  npm run import-g4m-csv           (domyślnie 300 URL-i na raz)
  npm run import-g4m-csv 1000      (lub podana liczba)

  Sprawdza co już jest w bazie — bezpieczny do wielokrotnego uruchamiania.

-- Metoda B: ze sitemapę G4M (aktualizacje, nowe produkty) --

  npm run import-g4m               (domyślnie 20 nowych URL-i)
  npm run import-g4m 500           (lub podana liczba)

  Pobiera sitemapę przez Playwright, wstawia tylko URL-e których nie ma w bazie.

Po każdym imporcie:
  npm run pipeline

4. RIFF.NET.PL
==========================================

! UWAGA: Riff.net.pl ma rate-limiting i Cloudflare Turnstile.
  Importuj małymi partiami z przerwami między sesjami.

-- Metoda A: strony kategorii (8 kategorii) --

  npm run import-riffnet           (wszystkie strony kategorii)
  npm run import-riffnet 2         (max 2 strony na kategorię — bezpieczniej)

  Kategorie: gitary elektryczne, akustyczne, klasyczne, basowe,
             wzmacniacze x3, efekty.

  Znany bug: AJAX paginacja zwraca duplikaty od strony 2+.
  Rate-limit aktywuje się po ~3 stronach per sesja.

-- Metoda B: sitemap XML (z rotacją proxy) --

  npx ts-node scripts/importRiffNetSitemap.ts 50   (50 produktów)

  Używa rotacji 10 proxy datacenter (Webshare).
  Cloudflare Turnstile blokuje datacenter IP na stronach produktów —
  sitemap pobiera się bez problemu, scraping pełnych stron może być blokowany.

Praktyczna strategia:
  - Małe partie (import-riffnet 1 lub 2) co kilka godzin
  - ~40–80 produktów dziennie przy ostrożnym podejściu

Po każdym imporcie:
  npm run pipeline
